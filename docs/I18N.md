# Bilingual Strategy

This repository uses one Skill and one repository for both Chinese and English users.

## Why Not Two Repositories?

Two repositories would split:

- stars
- issues
- examples
- evals
- releases
- community feedback
- maintenance effort

The Skill's core behavior is the same in both languages, so a single repository is simpler and stronger.

## How Confusion Is Avoided

The Skill uses language routing:

- Chinese prompt -> Chinese response -> Chinese references
- English prompt -> English response -> English references
- mixed prompt -> follow the language of the product idea itself

User-facing responses should not mix Chinese and English unless requested.

## File Layout

```text
product-idea-excavator/
  SKILL.md
  references/
    *.md          # Chinese references
    en/*.md       # English references
```

## Documentation Strategy

- Root `README.md`: English-first, with a clear Chinese entry.
- `docs/README.zh-CN.md`: Chinese overview.
- `docs/I18N.md`: explains language design.

## Maintenance Rule

When changing core behavior, update both:

- Chinese reference files
- English reference files

When changing only examples, update the language file that example belongs to.
