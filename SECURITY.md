# Security Policy

## Reporting Security Issues

Please do not open public issues for security-sensitive reports.

If this repository is published under an organization, use the organization's preferred private security contact.

If no private contact is listed yet, open a minimal issue saying that you have a security concern and request a private channel. Do not include exploit details in the issue.

## Scope

Security concerns may include:

- unsafe automation instructions
- prompt injection risk in bundled examples
- guidance that enables impersonation or deception
- data privacy oversights
- unsafe handling of public figures, voice, or identity
