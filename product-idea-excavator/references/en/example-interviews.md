# Example Interview Fragments

## Example 1: Vague AI Tool

User: I want to build an AI tool that helps teams be more productive.

Assistant:

That is still early, which is fine. "Productivity" is too broad to build from, so I see three possible wedges:

A. Meeting productivity: summaries, action items, follow-up tracking.  
B. Customer feedback productivity: turn scattered feedback into product insights.  
C. Internal knowledge productivity: help employees find company answers faster.

My product instinct is that B has a clearer business outcome because it can improve product decisions.

First question: what specific inefficient team moment made you think of this tool?

## Example 2: Too Many Features

User: I want to build a customer management system with customer lists, chat, AI summaries, contracts, payments, tasks, and dashboards.

Assistant:

This is already close to a full CRM. If v1 tries to do all of it, we will not know which value actually works.

I see three different products inside this:

A. Sales CRM: leads, follow-up, closing.  
B. Customer success tool: renewal, risk, service history.  
C. Customer insight tool: feedback, needs, product opportunities.

My recommendation is to choose one core workflow first.

Next question: are you trying to help sales win new customers, or help product/customer-success teams understand existing customers?

## Example 3: AI Agent

User: I want an AI Agent that automatically operates my social media account.

Assistant:

This has potential, but "automatic operation" is risky because it touches content generation, publishing, replies, account safety, and brand risk.

I would not start with fully automated publishing. A safer MVP is "generate suggestions -> user confirms -> semi-automated execution." That builds trust and reduces downside.

Next question: which step should v1 solve first: topic selection, writing, image generation, scheduling, replies, or performance review?
