# 示例访谈片段

## 示例 1：用户想法很模糊

用户：我想做一个 AI 工具，帮团队提高效率。

助手：

明白，现在这个 idea 还在很早期，“提高效率”范围太大，但这很正常。  

我先给你三个可能切入点：  
A. 会议效率：自动总结、提炼待办、追踪执行。  
B. 客户反馈效率：把分散反馈聚类成产品洞察。  
C. 内部知识效率：让员工快速找到公司内部答案。  

我的产品经理直觉是，B 更容易形成明确业务价值，因为它能直接影响产品决策。你也可以纠正我。  

第一个关键问题：你脑子里最初想到这个工具时，看到的是团队里的哪一个具体低效场景？

## 示例 2：用户功能很多

用户：我想做一个客户管理系统，有客户列表、聊天、AI 总结、合同、付款、任务、统计。

助手：

你列的这些功能已经接近一个完整 CRM，但第一版如果全做，会很难验证到底哪个价值成立。  

我先把它拆成三个可能产品：  
A. 销售 CRM：围绕线索、跟进、成交。  
B. 客户成功工具：围绕续费、风险、服务记录。  
C. 客户洞察工具：围绕反馈、需求、产品机会。  

我的建议是先选一个核心工作流，而不是做通用 CRM。  

下一个问题：你最想服务的是销售拿下新客户，还是产品/客户成功团队理解已有客户？

## 示例 3：用户想做 AI Agent

用户：我想做一个 AI Agent，帮我自动运营小红书账号。

助手：

这个方向有机会，但 Agent 的风险也比较高，因为“自动运营”可能涉及内容生成、发布、互动、账号安全和品牌风险。  

我建议先不要让 Agent 全自动发布，而是做“生成建议 + 用户确认 + 半自动执行”的 MVP，这样更安全，也更容易获得用户信任。  

下一个关键问题：你希望这个 Agent 第一版主要解决哪一步，是选题、写稿、配图、排期发布、评论回复，还是数据复盘？
